# Задача: Веб-приложение Elo-рейтинга для пляжного волейбола

## Описание проекта
Создать веб-приложение для расчёта и отображения Elo-рейтинга игроков в пляжный волейбол (2×2). Рейтинг рассчитывается индивидуально для каждого игрока на основе результатов матчей между парами. Пары могут меняться от игры к игре.

## Технический стек
- **Сборщик**: Vite 5.x
- **Frontend**: Vanilla JavaScript (ES6+), нативный CSS
- **Хранение данных**: JSON-файлы (плоские файлы)
- **Без фреймворков** — чистый JS

## Исходные данные
Результаты матчей предоставлены в структурированном JSON-формате. Каждая строка таблицы — это один матч (один сет = один матч). Победитель определяется по счёту (team1 выиграла, если score[0] > score[1]).

## Алгоритм Elo-рейтинга (по модели Lichess)

### Базовые параметры:
- **Начальный рейтинг**: 1500
- **K-фактор**: 32 (позже можно сделать динамическим в зависимости от рейтинга)
- **Калибровочный период**: первые 15 игр — повышенный K-фактор (например, 40)

### Расчёт для командной игры 2×2:
1. Для матча рассчитывается **средний рейтинг каждой команды** (среднее арифметическое рейтингов двух игроков)
2. Ожидаемый результат: `E = 1 / (1 + 10^((R_opponent - R_player) / 400))`
3. Изменение рейтинга: `ΔR = K * (S - E)`, где S = 1 (победа) или 0 (поражение)
4. Каждый игрок получает своё изменение рейтинга на основе среднего рейтинга команды-соперника

### Важно:
- Разница в счёте (15-3 vs 15-14) НЕ влияет на расчёт — только факт победы/поражения
- Матчи обрабатываются последовательно в хронологическом порядке (по id внутри группы)

## Требования к интерфейсу

### Главная страница: Таблица рейтинга
- Список всех игроков, отсортированный по рейтингу (убывание)
- Колонки: Место | Имя | Рейтинг | Игр сыграно | Тренд (↑↓)
- Индикатор калибровки (иконка или маркер для игроков < 15 игр)
- Строка кликабельна — открывает профиль игрока

### Профиль игрока (модальное окно или отдельная страница)
- Имя, текущий рейтинг, количество игр
- **Топ-10 последних матчей** с информацией:
  - Дата
  - Напарник
  - Соперники (пара)
  - Результат (победа/поражение, счёт)
  - Изменение рейтинга (+16 / -12) с цветовой индикацией (зелёный/красный)
- **График истории рейтинга** (линейный график, ось X — дата/номер игры, ось Y — рейтинг)

### Дополнительные требования к UI:
- Тёмная тема (dark mode) — основные цвета:
  - Фон: #1a1a2e или #16213e
  - Карточки/блоки: #1f2937 или #0f3460
  - Текст: #e4e4e7
  - Акцент: #3b82f6 (синий)
  - Победа/рост: #22c55e (зелёный)
  - Поражение/падение: #ef4444 (красный)
- Адаптивная вёрстка (mobile-first)
- Плавные анимации при переходах
